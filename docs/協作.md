## 3. 專案架構與重構建議 (Refactoring Plan)

目前的專案結構（按分層架構：`presentation/`, `domain/`, `data/`）在初期非常乾淨。但隨著 Phase 6 將引入複雜的「訓練週期」、「智慧排表」等龐大領域知識，這種結構會讓**相同功能的檔案散落各地**（例如改一個 Training 功能，要同時開五個不同資料夾的檔案）。

**建議作法**：在 Phase 6 開始前，將架構重構為 **Feature-First (依功能分組)**，例如：
- `lib/features/auth/` (放 LoginPage, AuthBloc, AuthRepo)
- `lib/features/cycle/` (放 Cycle 模型, 演算法, UI)
- `lib/features/training/` (放 執行訓練的邏輯與 UI)
-保留共用的 `lib/core/` 與共用元件 `lib/widgets/`。

## 4. 待定義項目與技術邊界建議 (To Be Defined & Boundaries)

為了確保 App 運作流暢且便於維護，建議採用以下分級 **數據管理邊界與儲存方案 (Storage Strategy)**：

| 數據/模組類型 | 說明與邊界 | 推薦儲存/實作方式 | 方案比較與原因 |
| :--- | :--- | :--- | :--- |
| **App 設定檔** | 主題顏色、語言、首頁排版偏好 | **維持使用 `Hive`** | **SharedPreferences vs Hive**:<br>- **Shared Preferences**：基於原生的簡單 Key-Value 存儲，只支援基本資料型別，讀寫速度普通。<br>- **Hive**：純 Dart 編寫的 NoSQL 資料庫，支援複雜物件，**讀取速度極快（全在 Memory Cache）**。既然專案已引入 Hive，建議統一使用 Hive 存放設定，效能更好且不需額外依賴。 |
| **系統預設模板** | 龍舟週期表、預設課表等「核心 Know-How」 | **`Dart Class`** | **Dart Class vs JSON**:<br>- **JSON**：適合從網路下載或動態更新，但需寫額外的解析 (Parsing) 程式碼。<br>- **Dart Class**：因為這些專業知識 (Domain Know-How) 是跟著 App 版本走的，直接寫成 `const List<Workout>` 的 Dart 實體類別，**效能最好、強型別有保障、且不用寫非同步的讀取邏輯**。 |
| **大量訓練資料** | 使用者自訂週期、TCX 檔、Laps 數據 | **`SQLite` (透過 `sqflite`)** | 支援關聯式查詢（例：「查出過去一個月指定心率的訓練」），擴充性高且處理大量歷史數據效率佳。 |
| **智慧生成演算法** | 多專項混合、自動排表邏輯 (Cycle Generator) | 初期 **Local Rule Engine**<br>未來轉移至 **Cloud Functions** | 初期將邏輯寫在 App 內成本最低。未來若演算法變複雜，需轉移至雲端以減輕 App 體積，並保護核心演算法不被反編譯。 |

### 未來轉換至 Cloud Functions (Serverless) 的預備工作
如果在未來（Phase 5 之後）決定將「智慧課表生成」演算法移至雲端（例如：Firebase Cloud Functions / AWS Lambda），我們需要做以下準備：
1. **API 規格定義**：定義 App 傳送給雲端的參數（使用者成績、偏好項目）與雲端回傳的 JSON 結構（自動生成的課表）。
2. **網路狀態處理**：App 需要實作「離線建立週期」的佔位符 (Loading/Offline state)，因為生成依賴網路。
3. **雲端架構搭建**：在 GCP 或 AWS 上撰寫 Node.js/Python 的微服務，將運動科學演算法從 Dart 移植過去。
4. **身份驗證綁定**：確保只有合法的授權使用者（透過 Google/LINE 登入的 Token）能呼叫這個雲端 API。

### 追加功能建議：Google 雲端硬碟 (Google Drive) 備份
- **實作方式**：串接 Flutter 的 `googleapis` 與 `google_sign_in`。
- **應用場景**：使用者在 App 內登入 Google 帳號後，系統可於背景自動將 SQLite 資料庫備份至其 Google Drive 的「專屬應用程式設定資料夾」（對使用者隱藏）。
- **優勢**：伺服器零成本（佔用使用者私有雲端空間），且換設備時可無縫還原歷史活動與自訂課表。
