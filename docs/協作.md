## 3. Collaboration Standards & Verification (協作規範與驗證)

### 3.1 Development Workflow (開發規範)
- **Feature Development**: All new features must update SA/SD status ([Implemented] / [In Progress] / [To-be-Developed]).
  - **功能開發流程**：所有新功能須先在 `SA/SD` 進行影響分析與狀態更新。
- **Coding Standard**: Follow `flutter_bloc` pattern and Feature-First directory (post-Phase 6).
  - **代碼規範**：遵循 `flutter_bloc` 狀態管理模式與 Feature-First 目錄規範。
- **Version Control**: Update this `協作.md` after major phase completions.
  - **版本控制**：重大 Phase 完成後應更新此檔案。

### 3.2 Verification Guide (驗證指引)
- **Auth (Login State)**: Stored in `auth_persistence` Box, key `current_user`.
- **Settings (Preferences)**: Stored in `settings` Box, key `current_config`.
- **Manual Verification**: Check Chrome DevTools -> Application -> IndexedDB.

## 4. Architecture & Refactoring Plan (專案架構與重構建議)
> [!NOTE]
> This section will be re-analyzed and organized after completion of Phase 5.
> 本段落將於 Phase 5 完成後進行再次分析整理。

The current layered structure (`presentation/`, `domain/`, `data/`) is clean for early stages but will scatter logic as we add complex Training Cycle features.
目前的分層架構在初期乾淨，但隨功能增加會導致相同功能的檔案散落各地。

**Feature-First Architecture (Implemented in Phase 6)**:
- `lib/features/auth/`: Profile and Authentication logic.
- `lib/features/settings/`: App configuration and theme control.
- `lib/features/activities/`: Activity recording, history, and TCX parsers.
- `lib/features/training/`: Training cycles, real-time HUD, and physics performance.
- `lib/features/home/`: Unified dashboard.
- `lib/core/`: Bottom-level services, enums, and theme templates.
- `lib/shared/`: Global UI elements and navigation.

### 5.1 Cloud Functions Transition (Serverless 預備工作)
Key steps for migrating complex algorithms to the cloud:
1. **API Specification**: Define parameters (performance, preferences) and return JSON structure.
2. **Offline Handling**: Implement placeholders/loading states for offline cycle generation.
3. **Cloud Architecture**: Node.js/Python microservices on GCP/AWS.
4. **Auth Binding**: Secure APIs with Google/Line OAuth tokens.

### 5.2 Google Drive Backup Plan (雲端備份計畫)
- **Implementation**: Sync `googleapis` with `google_sign_in`.
- **Scenario**: Background backup of SQLite DB to private "App Data Folder".
- **Benefit**: Server-less cost and seamless device migration.
