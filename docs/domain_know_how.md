# 領域知識分析：運動科學與訓練週期 (Domain Know-How)

本文件旨在梳理 `dptapp` (龍舟性能助手) 在 **科學化訓練管理** 方面的核心領域知識，確保應用程式的架構能真實反映專業運動員與教練的需求。

## 1. 訓練架構階層 (Training Hierarchy)

系統的訓練結構遵循現代運動科學的週期化訓練（Periodization）原則，分為以下階層：

1. **專項 (Discipline / Specialization)**：例如：龍舟 (Dragon Boat)、划船 (Rowing)、SUP 等。
2. **訓練週期 (Cycle - Macro/Meso)**：
   - 針對特定專項所設計的長/中程訓練目標。
   - 範例：「途中速度週期 (Mid-race Speed Cycle)」、「有氧耐力打底週期 (Aerobic Base Cycle)」。
   - 如果使用者同時跑數個專項或週期，系統會透過內建的交叉訓練邏輯，混合並調配出最適合的課表以避免過度訓練 (Overtraining)。
3. **訓練課表 (Workout / Schedule - Micro)**：
   - 週期內具體的單次訓練任務。有預設的子階段 (Phases)。
4. **分段/計圈 (Laps / Splits)**：
   - 課表內的最小執行單位，包含目標強度與持續時間。

## 2. 週期與訓練課表自動生成邏輯
由於之後有想要擴充成多專項運動的訓練課表，所以先以龍舟為例，但實際的邏輯會是通用的。且需要可以讓使用者自行添加項目、週期、課表等。

週期模板:
1. 全年週期: 假設12月比賽為主要賽事，則1~4月全部是力量週期，5~8月是途中速度週期，9~11月是專項轉換(視專項賽事而定，通常是長距離)週期，12月是專項模擬(減量)週期。
2. 三個月週期: 假設這個比賽剛結束，下一個比賽在三個月後，則直接選擇兩週恢復的力量週期，六週的途中速度週期，三週的專項轉換(長距離)週期，一週的專項模擬(減量)週期。
3. 多專項混和週期: 假設使用者同時跑耐力型專項(鐵人三項或馬拉松等)和爆發型專項(舉重、球類運動)或混合型運動(龍舟、crossfit等)，則系統會根據項目的優先順序、個人運動表現自動生成並帶出該週期的每週(每日)課表。

以上週期皆只是範例，實際的週期會根據使用者的個人狀況而有所不同而調整。
基本的運動科學邏輯是: 
1. 力量週期 -> 途中速度週期 -> 專項轉換(長距離)週期 -> 專項模擬(減量)週期。
2. 各週期體能檢測項目:
力量-體重換算健力三項成績
爆發(途中最高速度)-體重換算舉重(抓舉、挺舉、高翻、高拉)成績
徒手爆發-短跑、單槓
專項轉換(以龍舟為例)-划船機最高速度、划船機60秒輸出、划船機120秒輸出
專項模擬-單人龍舟、模擬隊形
3. 依照個人成績、專項優先序或個人偏好，自動生成並帶出該週期的每週(每日)課表。


當使用者設定當前週期為 **「途中速度」週期** 時，系統會自動生成並帶出該週期的每日課表。以其中一天的課表為例：

### 課表結構範例 (途中速度週期)

| 階段 (Phase) | 圈數 (Lap / Rest) | 目標強度 | 持續時間/距離 | UI 提示與動作 |
| :--- | :--- | :--- | :--- | :--- |
| **熱身 (Warm-up)** | Lap 1 | 60% | 2 分鐘 | 開始計時，顯示目標配速 |
| | Rest | 恢復 (0%) | 2 分鐘 | 倒數 5 秒提示音 (準備下一圈) |

## 3. 實際執行與 UI 互動 (Training Execution)

在 `Training` 畫面實際執行該課表時：

- **時間軸與提示 (Timeline & Cues)**：
  - 啟動課表後，主時間軸開始記錄。
  - UI 畫面上方會顯示類似「音樂 APP 歌詞」的滾動式提示 (Scrolling Workout Cues)，讓運動員提前預知下一個 Lap 的強度與剩餘時間。
- **階段提醒 (Alerts)**：
  - 在每個階段轉換（例如：高強度轉為組休）前 3-5 秒，系統發出倒數音效或震動，以彌補水上運動難以頻繁看手錶/手機的痛點。
- **計圈與停錶 (Laps & Stops)**：
  - 支援 **自動給予時間戳記** (根據預設計畫時間自動切換 Lap)。
  - 支援 **手動點擊** 計圈或暫停 (適用於水流變化或突發狀況導致的時間延遲)。
- **賽後調整 (Post-Workout Adjustment)**：
  - 實際的 Laps 數據（如果手動切換時間有誤差），可以在訓練結束後，進入 `Activities` > `Activity Detail` 頁面中手動微調切割點。

## 4. UI/UX 與導航結構調整計畫

為了融入上述強大的 Cycle 系統，App 的導航與版面需進行以下調整：

### A. Footer (Bottom Navigation Bar) 改版
由於空間有限，「Cycle」將成為核心功能，取代獨立的 Activities。
- **新結構**：`Home` | `Cycle` (內含 Activities) | `Training` | `Community` (包含勳章/里程碑) | `Settings`
- **Cycle 頁面職責**：安排未來的訓練課表、檢視當前進度、並向下鑽取 (Drill-down) 檢視過去已完成的 **Activities**。

### B. Header 與 Sidebar (Drawer) 定位重構
- **Header**：
  - 新增 **User Avatar (頭像)**：點擊可快速進入 Profile。
  - 新增 **Notification (通知鈴鐺)**：顯示好友動態、挑戰邀請或訓練提醒。
- **Sidebar (漢堡選單)**：
  - 統一美術風格（與 Elite Dragon 深/淺色主題一致）。
  - 定位：作為全局設定、設備連接狀態、以及非核心輔助功能（如：匯出報告總管、關於我們）的收納處，不再與 Footer 的核心導航重疊。

### C. 首頁 (Home) 視覺與客製化
- **色彩統一**：修正目前首頁淺色/Default 顏色不一致的問題，使其與全局主題（深色運動風）統一。
- **客製化 Widget**：Home 畫面改為卡片式儀表板。
- **預設顯示**：首頁預設展示「當前週期狀態」，包含圓餅圖或進度條顯示「已完成課表」與「未完成課表」，讓使用者一開 App 就知道今天要練什麼。

## 5. 領域專業知識的封裝 (Encapsulation)

**將專業領域知識（如：龍舟訓練週期、課表模板）直接封裝在 App 中，是此產品的核心價值。**

- **存放方式**：系統預設的模板不需要放在外部資料庫，可以直接寫在 App 內部的 `assets/json` 或 Dart 的實體類別（如 `DomainTemplates` class）中。
- **運作機制**：
  - **預設模板 (Built-in)**：當使用者選擇「龍舟 - 途中速度週期」時，系統直接從內部唯讀讀取。隨 App 版本更新而擴充。
  - **使用者自定義 (Custom)**：如果使用者想要微調預設模板（例如：修改秒數或強度），系統會「複製」一份預設模板，並存入使用者的本機資料庫（SQLite）中，成為專屬的「自訂模板」。
- **商業價值**：未來若有付費解鎖高階課表的需求，只需透過內購系統解鎖這部分靜態檔案的存取權。
